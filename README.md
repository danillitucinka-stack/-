## Шаг 1: Установка зависимостей

Откройте терминал и выполните:

```bash
pip install -r requirements.txt
```

## Шаг 2: Настройка IP-адреса

Перед запуском нужно настроить IP-адрес сервера в клиенте:

1. Откройте файл `client/client.py`
2. Найдите строку: SERVER_IP = "192.168.1.100"
3. Замените на IP-адрес вашего ПК (сервера)

**Как узнать свой IP:**
- Откройте командную строку (Win + R, введите cmd)
- Введите: ipconfig
- Найдите строку "IPv4-адрес" (например, 192.168.1.15)

## Шаг 3: Запуск

### Вариант А: На одном компьютере (тестирование)

1. Запустите сервер: `python server/server.py`
2. Запустите клиент: `python client/client.py`

### Вариант Б: На разных компьютерах (реальное использование)

Компьютер помощника (сервер):
1. Запустите `python server/server.py`
2. Убедитесь, что брандмауэр разрешает соединения на порт 9999

Компьютер, которому нужна помощь (клиент):
1. Скопируйте папку client на этот компьютер
2. Измените SERVER_IP в client.py на IP сервера
3. Запустите `python client/client.py`

# ФУНКЦИОНАЛ
# =============================================================================

### Remote View (Просмотр экрана)
- Клиент автоматически делает скриншот каждые 5 секунд
- Сервер отображает скриншоты в реальном времени
- Вы видите всё, что происходит на экране другого ПК

### File Explorer (Файловый браузер)
- На сервере: кнопки навигации "Вверх", "Домой", "Обновить"
- Двойной клик по папке - переход внутрь
- Двойной клик по файлу - информация о файле

### Auto-Start (Автозапуск)
- При первом запуске клиент добавляет себя в автозагрузку Windows
- После перезагрузки компьютера программа запустится автоматически
- Папка автозагрузки: %APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup

### Ошибка: "Connection refused"
- Проверьте, что сервер запущен
- Проверьте правильность IP-адреса в client.py
- Проверьте, что порт 9999 не заблокирован брандмауэром

### Ошибка: "No module named 'pyautogui'"
- Выполните: pip install pyautogui Pillow

### Ошибка: "Permission denied" при автозапуске
- Запустите клиент от имени администратора

### Скриншоты не отображаются
- Проверьте, что клиент и сервер в одной сети
- Убедитесь, что на клиенте запущен Python с правами администратора

### Сетевой протокол
- TCP (Transmission Control Protocol)
- Порт: 9999
- Тип соединения: клиент-сервер

### Формат данных
- Скриншоты: JPEG, base64-кодирование
- Команды файлового браузера: текстовые (LIST:путь, INFO:путь)

### Библиотеки
- socket - работа с сетью
- pyautogui - скриншоты
- PIL/Pillow - обработка изображений
- tkinter - графический интерфейс
- threading - многозадачность
